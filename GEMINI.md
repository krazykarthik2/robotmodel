# Mini π0.5 / SmolVLA for a 4-DOF Robot: Project Master Plan

## 0. Fixed Constraints
* **Dataset:** BridgeData V2 (Raw data allowed).
* **Preprocessing:** Preprocessing (Image-to-Embedding, Tokenization, Action-to-Trajectory) is enabled as part of the pipeline.
* **Input State:** Current end-effector (EEF) state [x, y, z, g].
* **Output:** Future relative EEF trajectories (16 waypoints).
* **Hardware:** 2× NVIDIA L4 GPUs (Note: Multi-GPU currently restricted by NCCL/Driver mismatch; fallback to single-GPU is implemented).
* **Precision:** BF16 (Optimized for L4) with **FP32 Loss Calculation**.
* **Optimizer:** Muon + AuxAdam. We use Muon for the 2D weights in the transformer backbone (layers) to maintain orthogonality, while using AdamW for embeddings, norms, output heads, and custom projectors for better stability and precision in those components. This ensures optimal gradient flow through the main backbone while preserving the sensitivity of the input/output layers.

## 1. Vision & Language Pipeline (Native SmolVLM)
* **Base Model:** `HuggingFaceTB/SmolVLM-256M-Instruct` (Idefics3 Architecture).
* **Vision Encoder:** Native SmolVLM vision tower (`.model.vision_model`).
* **Modality Aligner:** Native SmolVLM multi-modal projector (`.model.connector`). Features pixel shuffling to compress high-res grid patches into a dense, shared 576-dim language space.
* **Text Encoder:** Native SmolVLM Tokenizer and Embeddings (`.model.text_model.get_input_embeddings()`).
* **Offline Preprocessing:** Images pass through the native vision tower and connector to be saved as compressed visual tokens in Parquet, preventing disk bloat.

## 2. Model Architecture (SmolVLA)
* **Backbone:** Native SmolVLM Language Model (`.model.text_model`).
* **Freezing:** Native parameters frozen. LoRA applied to language model `q_proj`, `v_proj`, `k_proj`, `o_proj`.
* **LoRA:** Rank 32, Alpha 64, Dropout 0.05.
* **Token Sequence:** `[VISION TOKENS] [STATE TOKEN] [LANGUAGE TOKENS]`
    * Vision: Compressed tokens from the native connector or SigLIP embeddings projected via `vision_proj`.
    * State: 1 token (Projected from 4-dim EEF via Custom MLP to 576-dim).
    * Language: Variable length instruction embeddings.
* **Output Head:** **Flow Head** (Conditional Flow Matching). Predicts the velocity field $v(x_\tau, \tau)$ given the hidden state and current noisy action $x_\tau$ at time $\tau$.

## 3. Training & Normalization Details
* **Data Scaling:** **Quantile Normalization** (1st/99th percentiles mapped to $[-1, 1]$). Robust to outliers and consistent across all action dimensions (x, y, z, gripper). Stats stored in `norm_stats.json`.
* **Loss Function:** **Conditional Flow Matching (CFM)**. MSE loss between predicted velocity and target velocity $(x_1 - x_0)$ in **FP32**.
* **Generation:** **Euler Integration**. Trajectories are generated by integrating the velocity field from $\tau=0$ to $\tau=1$ starting from Gaussian noise.
* **Default Training Mode:** "Overfit" (single-sample mastery).
* **Batch Size:** 128.

## 4. Execution Pipeline & Simulation
* **Inference:** Camera -> SmolVLM AutoProcessor -> SmolVLM Vision -> SmolVLA -> 16-step EEF Trajectory.
* **Simulation:** PyBullet with a **Floating End-Effector**. URDF and Inverse Kinematics (IK) are completely bypassed. The model directly controls the X/Y/Z translation of a floating gripper box.
* **Grasping Logic:** Proximity-based dynamic weld constraints upon gripper closure.
* **Camera:** Statically aligned to mimic the Bridge V2 (WidowX) dataset perspective.
